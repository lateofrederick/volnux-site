Architecture

A mesh that thinks
for itself.

Volnux replaces the central scheduler with a living network. Nodes discover each other, negotiate execution, and reroute on failure — without any coordination server.

01

Node Discovery

New nodes broadcast their presence over the mesh. Existing peers respond with topology metadata. No registry. No seed server. The mesh self-assembles.

02

mTLS Handshake

Every link is mutually authenticated before a single byte of payload crosses. Certificate rotation happens live without dropping connections.

03

Task Dispatch

The runtime reads node and executor annotations from your Pointy-lang workflow and routes each task to the correct peer or Celery worker automatically.

04

Failure Recovery

The watchdog detects dead links in real time. Failed tasks rehydrate from their last checkpoint on any available node. No human intervention required.

Capabilities

Built for real
distributed systems.

Every component designed from first principles for production mesh operation.

⬡

Peer-to-peer topology

No master node. No coordinator process. Every Volnux node is a full peer. The mesh routes, balances, and heals without centralised coordination.

⇄

Adaptive mini-context

Execution state switches between small, batch, and aggregate modes automatically. Large workloads stream their results without memory pressure on any node.

◎

Checkpoint rehydration

Every task is checkpointed at execution boundaries. A failed node's work rehydrates on any healthy peer. Workflows never restart from zero.

⚡

Multi-executor dispatch

Route tasks to mesh nodes, Celery workers, or local processes — all from Pointy-lang annotations. No custom routing code required.

⬤

mTLS by default

Every inter-node connection is mutually authenticated. Certificate rotation happens live. Zero-trust networking with no extra configuration.

◈

OTEL telemetry

Every packet, every state transition, every retry is traced with OpenTelemetry. Full distributed trace across the entire mesh without instrumentation overhead.

Dispatch model

Your infrastructure.
Your annotations.

Declare where each task runs directly in Pointy-lang. The mesh handles the rest — routing, retrying, and streaming results back.

Dispatch to any peer in the cluster by node ID

local

GPU Cluster

Pin compute-heavy AI tasks to GPU-equipped peers

node="gpu-cluster"
