Publishing

Your events.
The whole ecosystem.

Any Python class that inherits from EventBase can be published to EventHub. Once published, it becomes instantly referenceable in any Pointy-lang workflow across the entire Volnux mesh — no deployment, no shared codebase, no coordination overhead.

01
Inherit EventBase

Subclass EventBase and implement the __call__ coroutine. That's the entire interface contract.

02
Publish to PyPI, Git, or EventHub

Push to your preferred source. EventHub picks up verified publishers automatically.

03
Reference by name in Pointy-lang

Anyone on the Volnux mesh can now pull and execute your event with a single annotation.

Publish your first event →
my_event.py
# 1. Inherit EventBase
from volnux.events import EventBase
from volnux.result import EventResult

@hub.publish(name="MyTransform", version="1.0.0")
class MyTransform(EventBase):
  async def __call__(self, record: dict) -> EventResult:
    # your logic here
    result = await self.process(record)
    return EventResult(content=result)
pipeline.pointy
# 3. Reference anywhere in Pointy-lang
hub:MyTransform@v1.0.0
  -> pypi:SnowflakeLoad@v1.0.4

How it works

Runtime assembly, not deployment.

01

Author once

Write a Python class that inherits EventBase and implement a single async coroutine. No framework lock-in, no decorators, no boilerplate beyond the interface contract.

02

Publish anywhere

Push to PyPI, tag a Git release, or register on EventHub. Version pins ensure reproducible execution. The Volnux resolver fetches exactly the version you specify at runtime.

pypi:MyEvent@v2.1.0
git:OrgEvent[version="v3.1"]
hub:CommunityEvent@latest

03

Run without deployment

The Volnux runtime resolves and fetches your event at execution time — on any node in the mesh. No shared codebase. No deployment pipeline. No coordination. Reference it in Pointy-lang and it just runs.
